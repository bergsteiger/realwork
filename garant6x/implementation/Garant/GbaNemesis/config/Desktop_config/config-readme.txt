        Файл определения настроек ГАРАНТ 6

   (!) текуший эталон файла конфигураций находится внутри файла: "..\..\nsSettings.pas"

   Файл состоит из двух секций: определения свойств и определения
 конфигураций. Первая содержит описание групп и свойств, вторая -
 значений свойств в различных конфигурациях.

   Свойства объединены в группы, и представляют собой иерархическую
 структуру. На верхнем уровне - группы, на следующих - свойства, описания
 одних свойств могут быть вложены в другие, например:

group "PageSetup" {        // 'group' и 'property' - ключевые слова,

  property "Page" {
    ...
    property "Margin" {
      ...
    }
  }

  property "Printer" {
    ...
    property "Paper" {
      ...
    }
  }
}

group "Databases" {
  ...
}

...


   Свойство характеризуется рядом параметров, в приведенном ниже примере
 разъясняется формат их определения:

group "PageSetup" {
  property "Page" {                                  // "Page" - идентификатор свойства
    name: "Настройки страницы";                      // не обязательный, по умолчанию: ""
    hint: "Позволяют изменить внешний вид страницы"; // не обязательный, по умолчанию: ""
    tags: visible, unique, constant;                 // не обязательный, по умолчанию: hidden, not_unique, variable
    constraint: { FREE : boolean };                  // задает тип свойства: FREE, PATTERN, LIST или LIST_MAY_EMPTY
                                                     // и ограничения: для списков - это набор значений, для
                                                     // 'FREE' - тип значения, для шаблона - последовательность типов

    // constraint: { LIST /* или LIST_MAY_EMPTY */ : "Да", "Нет", "Не знаю" };
    // constraint: { PATTERN: boolean, number, string, date };

    property "Margin" {                              // вложенное свойство, его полный идентификатор:
      name: ...                                      // "PageSetup/Page/Margin"
    }
  }
}

   В случае если само свойство или одно из вложенных свойств имеет tag
 visible, то атрибуты name и hint ОБЯЗАНЫ быть заданными, не смотря на то,
 что формально они являются необязательными.


   Пример задания значений свойств в различных конфигурациях:

configuration "SMART" {                           // "SMART" - идентификатор конфигурации
  name: "Полная";
  hint: "Предоставляет максимальные возможности";
  tags: constant;                                 // не обязательный, по умолчанию: variable
  values {
    PageSetup/Page = true;
    PageSetup/Page/Margin = 1, "Left", "Top", false;
      ...
   }
}
 
configuration "SIMPLE" {
  name: ...
  hint: ...
  tags: ...
  values {
    PageSetup/Page/Margin = 2, "Left", "Top", true;

    // в этой конфигурации свойство PageSetup/Page
    // имеет такое же значение как и в SMART
  }
}

...
 

 Если для конфигурации значение по умолчанию не заданно, то оно берется
 из первой конфигурации.


  При задании значений параметров используются следующие типы данных:

 Тип            Значения
 --------------------------------
 boolean        true или false
 string         последовательность символов в двойных кавычках (пример: "ГАРАНТ 6")
 number         неотрицательное целое число (пример: 0, 10, 32256)
 date           (number, number, number) => соответственно (год, месяц, число)



   Язык описания настроек содержит ряд ключевых слов:

 boolean     hidden          NOT_VISIBLE   unique
 constant    hint            number        UNIQUE
 constraint  LIST            PATTERN       values
 date        LIST_MAY_EMPTY  property      variable
 FREE        name            string        visible
 group       not_unique      tags

 регистр написания ключевых слов нужно соблюдать: 'group' != 'Group'


   Описания настроек могут сопровождаться комментариями (в стиле С++):

      /* текст
         многострочного
         комментария
      */

      // текст однострочного комментария
