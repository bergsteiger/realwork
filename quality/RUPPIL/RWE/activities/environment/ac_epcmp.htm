Активность: Подготовить средства контроля за изменениями

Цель:
  • Задать и задокументировать процессы управления запросами на изменение для того, чтобы поток запросов не захлестнул поступательную работу по проекту и вместе с тем ни один из запросов не остался незамеченным, все запросы приняты на рассмотрение.
Входные артефакты:
  • Нет
 Результирующие артефакты:
  • Модель управления запросами на изменение
Роль: Менеджер по Качеству

Процесс контроля за изменениями состоит в том, чтобы собирать и анализировать поступающую информацию по изменениям. Для того, чтобы запросы не терялись с течением времени, чтобы собирать статистику по изменениям, а также в процессе обсуждать их, используется автоматизированное средство Rational ClearQuest.

Установление процесса контроля за изменениями состоит в том, чтобы разработать и реализовать как мини-проект схему, по которой будет работать Rational ClearQuest.

В дальнейшем, модель управления запросами на изменение может сама претерпевать различные изменения и совершенствоваться.

Менеджер проекта совместно с Менеджером по Качеству изначально разрабатывают модель, а специалист по автоматизированным средствам разработки ее реализует.

Текущий принятый в проекте вариант должен быть подробно описан и роздан для ознакомления всем членам команды разработки. Это позволит всем сотрудникам понимать жизненный цикл запроса.
