Роль: Менеджер Проекта

Задать "индикаторы"

"Индикаторы" проекта - это часть информации о проекте, дающая представление о том, насколько реальный прогресс проекта соответствует Плану Разработки ПО. По мере продвижения проекта, Менеджер Проекта контролирует его развитие путем просмотра этих индикаторов. Он может начать некоторые корректирующие процедуры, которые позволять исправить положение дел, когда ранее определенные условия не соблюдены (см. также . Установить процедуры и критерии действий по корректировке).
Возможны следующие индикаторы:

  • Пересмотренные границы проекта во времени (сделанная работа + ожидаемое завершение) в сравнении с запланированными.
  • Частота дефектов относительно заданных критериев качества
  • Индикаторы риска (ситуации, которые говорят о том, что риск реализовался)

Определение этих индикаторов зависит от бюджетов проекта, целей качества и планов (в частности, Плана разработки ПО). Эти индикаторы устанавливаются в Плане Измерений и Плане управления рисками.

Задать источники для индикаторов

В большинстве случаев индикаторы проекта вычисляются из более примитивных метрик, которые собираются в ходе проекта членами команды. Как это происходит описывается в Плане Измерений.

Раздел Мониторинг и контроль проекта Плана разработки ПО должен включать краткое описание индикаторов, используемых в проекте. Отдельные подразделы Плана разработки ПО относятся к планам, бюджету и качеству. Контроль проектных требований отдельно оговаривается в Плане Управления требованиями.

Установить процедуры для отчетности о состоянии

Кроме определения типов собираемых метрик и индикаторов проекта, необходимо установить процедуры и частоту отчетности. Эти процедуры должны описывать правила отчетности по завершению задач, достижение проектных milestone'ов. Также должны быть заданы все сведения для проведения обзоров Руководителем ОИТ или вышестоящим должностным лицом.

На основании данных по измерениям, Группа Качества формирует отчеты, в которых указывает состояние проекта. Далее, эти сведения Менеджер по Качеству анализирует и передает Руководителю ОИТ.

В случае необходимости Руководитель ОИТ дает руководящие указания по ведению проекта и корректировке усилий.

Установить процедуры и критерии действий по корректировке

Для обеспечения эффективного контроля над проектом, менеджер проекта задает границы (или триггеры) значений/условий для каждого из выбранных проектных индикаторов. Эти границы записываются в соответствующие подразделы Раздела Мониторинг и Контроль проекта Плана разработки ПО.

Когда границы превышены, Менеджер проекта может инициировать действия по корректировке, чтобы вернуть проект в нужное русло. В зависимости от конкретной ситуации, он может разрешить ситуацию, назначая необходимые дополнительные работы.
