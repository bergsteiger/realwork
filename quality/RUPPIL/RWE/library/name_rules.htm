Правила именования

Общие правила именования элементов модели

Все элементы именуются однотипно:

  • Имена элементов модели должны начинаться с прописной (большой) буквы.
  • Имя должно быть представлено существительным или существительным с дополнением, представляющим расширенное значение, в единственном числе.


    • Константы

  • Имя константы должно состоять из прописных (больших) букв.
  • Имена функций, состоящих из нескольких слов, должны именоваться с использованием символа подчеркивания (_), который разделяет такие слова. Каждое слово должно быть задано с использованием только прописных (больших) букв.

    • Пример: MAXIMUM, MAX_LONG.


    Перечисления

  • Имя элемента перечисления должно состоять из строчных (маленьких) букв.
  • Имена элементов перечислений, состоящих из нескольких слов, должны именоваться с использованием символа подчеркивания (_), который разделяет такие слова. Каждое слово должно быть задано с использованием только строчных (маленьких) букв.
  • Все элементы одного перечисления должны иметь префикс, который определяется по прописным (большим) буквам имени перечисления.
    • Пример: toc_red, toc_green для перечисления с именем TypeOfColor.


    Операции

  • Имя операции должно состоять из строчных (маленьких) букв.
  • Имена операций, состоящих из нескольких слов, должны именоваться с использованием символа подчеркивания (_), который разделяет такие слова. Каждое слово должно быть задано с использованием только строчных (маленьких) букв.
  • Для именования операций должны использоваться глаголы-инфинитивы или глагольные фразы.
    • Пример: Scheduler::run(), Scheduler::set_new_count().


    Параметры операций

  • Первый символ в имени параметра операции должен быть строчной (большой) буквой.
  • Для отображения уникальности или чтобы показать, что сущность является главным элементом действия, должны использоваться приставки the или this перед именем параметра операции.


    • Атрибуты классов

  • Имя переменной должно состоять из строчных (маленьких) букв.
  • Имена переменных, состоящих из нескольких слов, должны именоваться с использованием символа подчеркивания (_), который разделяет такие слова. Каждое слово должно быть задано с использованием только строчных (маленьких) букв.
  • Имена переменных-членов класса начинаются с префиксов m_.
  • Статические переменные класса начинаются с префикса s_.
  • Глобальные переменные начинаются с префикса g_.
    • Пример: m_count_of_lines, s_class_factory, g_object_manager.


    Атрибуты интерфейсов

  • Атрибуты интерфейсов должны именоваться так же, как операции.


    • Переменные и операции булевого типа

  • Название переменной или операции булевого типа должно начинаться с префиксов is или has.
  • Название переменной или операции булевого типа должно быть написано в положительном значении.
    • Пример: is_available, но не is_not_available.
