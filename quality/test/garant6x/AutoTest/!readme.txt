Инструкция по использованию утилиты автоматического тестирования:

1. Взять из CVS\quality\test\garant6x\AutoTest последнюю версию программы тестирования (EtalonTest.exe)
2. Там же взять пользовательские базы. Подложить их под сервер (или десктоп), если менялся формат баз, то необходимо залить XML.
3. Зайти в оболочку под пользователем Etalon - Etalon. При желани создать сложный запрос, по которому
найдётся документов 10-15 и сохранить его в папку под именем "Эталонный запрос"
4. В ту же директорию, где лежит утилита тестирования положить текущию версию библиотеки GblAdapter.dll
(клиентскую или десктопную). Запускать утилиту тестирования, передавая в качестве параметра № теста (номера в приложении), результаты
желательно выводить в файл, записать результаты (результаты - это время выполнения операции - отображено в логе работы утилиты, также результаты пишутся в файл: .\result.csv) 
в таблицу CVS\quality\test\garant6x\AutoTest\performance_result.xls.

5. ОБЯЗАТЕЛЬНО тесты по работе с настройками пускать в порядке 8, 9, 10!!!!!!!

Приложение 1. 
Номера тестов, передаваемые в качестве параметров утилите тестирования:
1 - тест загрузки структуры ГК
2 - тест загрузки текста ГК
3 - тест простого поиска
4 - тест сложного поиска
5 - тест фразального поиска
6 - тест получения иерархического списка
7 - тест получения списка всех документов
8 - тест создания новых настроек
9 - тест перезаписи новых настроек
10 - тест чтения новых настроек
11 - тест загрузки адаптера


Приложение 2.
Пример запуска утилиты тестирования для тестирования загрузки текста ГК с выводом результатов файл
C:\result.log:

EtalonTest.exe 2 1>C:\result.log 2>&1
