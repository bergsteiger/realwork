Глоссарий

1.     Введение

1.1     Цель

Этот документ представляет собой толковый словарь терминов, используемых при разработке Гарант 6.0.

1.2     Контекст

Создание документа преследовало цель предоставить всем участникам работ по разработке Гарант 6.0, единый толковый словарь терминов, используемых в работе.

1.3     Краткое содержание

Во втором подразделе (‘Определения’) даны термины и их толкования в алфавитном порядке.

Каждый термин, взятый из документов проектной документации Гарант 6.0, имеет соответствующую библиографическую ссылку.

1.4     Ссылки

Www-версия RUP доступна по адресу: http://10.14.3.29/RUP/

Информацию о терминах методологии RUP, по которой планируется вести разработку, можно получить в документе Внедрение Rational Unified Process. Глоссарий. (RUP.Glossary.rtf)

Документы, в которых дается более детальное описание терминов:

1. “Гарант 6.0. Положение о разработке”. (600.develop_case.rtf)

2. “Гарант 6.0. Видение”. (600.vision.rtf)

3. “Гарант 6.0. Запросы совладельцев”. (600.stakeholder_request.rtf)

2.     Определения

2.1     ACL

См. Access Control List (ACL)

2.2     Access Control List (ACL)

Список Контроля Доступа.

Список, содержащий информацию о пользователях, разграничивающий права их доступа к информации определенного вида в рамках рассматриваемой системы.

2.3     Wildcard

Групповой символ (заменяющий один или несколько символов: знак вопроса (?) может представлять любой одиночный символ; звездочка (*) используется для представления любого символа или группы символов).

2.4     WYSIWYG

Сокращение от англ. what you see is what you get – “то, что видишь, - это то, что получишь”. Термин применяется обычно по отношению к программам-просмотрщикам(редакторам) документов и означает, что увиденное на экране монитора будет так же выглядеть и в распечатанном на бумаге виде.

2.5     Аналогичный документ

Документ, у которого с рассматриваемым документом совпадают значимые атрибуты. См. Значимый атрибут документа

2.6     Атрибут типа Номер

Атрибут строкового типа, идентифицирующий документ. К таким атрибутам относятся различные номера документа, присвоенные ему государственными инстанциями.

2.7     Дельта

Файл, или совокупность файлов, содержащий данные для проведения пакетного пополнения системы

2.8     Задание интервалов (операция)

Логическое условие, указывающее, что отдельные слова, введенные в качестве параметра контекстного поиска, должны находиться в заданном интервале (например, в одном абзаце, предложении, на расстоянии не далее нескольких слов друг от друга и др.)

2.9     Значимый атрибут документа

Атрибут документа, который учитывается при проведении поиска по соответствию.

2.10     ИБ

См. Информационный блок (ИБ)

2.11     Информационный блок (ИБ)

Информационный блок – логическая единица информационного наполнения СПС Гарант, из совокупности которых составляется Комплект Информационных Блоков (КИБ).

Другое название – сегмент базы данных Гаранта.

2.12     Карточка запроса (КЗ)

Карточка запроса. Диалог, позволяющий пользователю задать все необходимые и доступные параметры поиска.

2.13     Карточка фильтров (КФ)

Диалог, позволяющий задать фильтрацию документов по определенным критериям. Тем самым сужается (“фильтруется”) множество документов, предоставляемых системой, согласно выбранному пользователем критерию фильтрования.

2.14     КЗ

См Карточка запроса (КЗ)

2.15     КИБ

См. Комплект информационных блоков (КИБ)

2.16     Ключевое выражение

Элемент словаря поиска по ситуациям.

2.17     Комплект информационных блоков (КИБ)

Комплект информационных блоков [Гаранта] – совокупность файлов, составляющая информационное наполнение продукта.

2.18     Корреспонденты [документа]

Документы, которые ссылаются на данный документ

2.19     КФ

См.Карточка фильтров (КФ)

2.20     ОМ

См. Основное меню

2.21     Основное меню

Диалог, содержащий ссылки для воспроизведения основных функций системы

2.22     Пакетное пополнение (ПП)

Процесс обновления системы (как информационной, так и программной ее частей)

2.23     Параметрический поиск

Поиск документов по любым атрибутам.

2.24     Патч

Совокупность файлов, позволяющих обновить программную часть системы.

2.25     ПЛС

См. Поиск лекарственных средств (ПЛС)

2.26     Поиск лекарственных средств (ПЛС)

Вариант поиска, результатом которого является список лекарственных средств.

2.27     Поиск по классификатору

Возможность осуществления пользователем навигации по древовидному классификатору документов.

2.28     Поиск по ситуации (ППС)

Частный случай поиска по реквизитам-ключевым словам, с собственным интерфейсом.

Уникальная поисковая возможность системы Гарант, которая позволяет находить документы, относящиеся к выбираемой пользователем предметной области, ключевым понятиям этой области и т.п.

2.29     Поиск по соответствию

Возможность поиска в документах норм права, регулирующих те же вопросы, что и текущий просматриваемый документ. При этом виде поиска учитываются значимые атрибуты документа.

2.30     Поиск по реквизитам (ППР)

Поиск документов по реквизитам (основным идентификационным атрибутам документов).

2.31     Поиск фармфирм (ПФФ)

Вариант поиска, результатом которого является список фармацевтических фирм.

2.32     Поисковый запрос

Совокупность параметров, заданных пользователем при проведении поиска.

2.33     Полный список атрибутов документа

·         Дата включения в версию

·         Утратил силу в версии

·         Регистрационный номер в МЮ

·         Дата регистрации в МЮ

·         Дата для сортировки

·         Дата анонсирования

·         Дата снятия с контроля

См. Пакетное пополнение (ПП)

См. Поиск по ситуации (ППС)

См. Поиск фармфирм (ПФФ)

2.38     Реквизит документа

Реквизитами документа являются следующие атрибуты документа:

·          

2.39     Реквизит-Словарь

Реквизит спискового типа (включая иерархические и одноуровневые разновидности списков), который может принимать значения из специально созданного для него Словаря.

2.40     Релевантность

Степень соответствия документа поисковым условиям. Определяется на основе количественного критерия соответствия заданных параметров поиска данному документу.

2.41     Респонденты [документа]

Документы, на которые ссылается данный документ.

2.42     РИБ

Региональный информационный блок. См. Информационный блок (ИБ)

2.43     Сервер пакетного пополнения (СПП)

Программа, реализующая часть механизмов пакетного пополнения, относящуюся к возможности получения дельт пользователями системы.

2.44     Следование (операция)

Логическое условие, задаваемое при контекстном поиске, указывающее, что порядок следования введенных для поиска слов должен учитываться при проведении поиска.

2.45     Словарь

Словарем называется перечислимое множество значений, которые может принять данный реквизит документа.

Обычно отдельный Словарь создается для каждого реквизита документа списочного типа. В этом случае Словарем также называют сам реквизит спискового типа, включая иерархические и одноуровневые разновидности списков.

2.46     Синонимический поиск

Поиск с учетом синонимов. При контекстном синонимическом поиске будут найдены элементы, содержащие заданный термин и его синонимы.

2.47     Сокращенный список атрибутов документа

Система показывает пользователю сокращенный список атрибутов документа при запросе пользователя получения информации о документе.

В сокращенный список атрибутов документа входят:

·         Название документа

·         Статус

·         Тип

·         Регистрационный номер документа

·         Дата регистрации документа

·         Регистрационный номер в МЮ

·         Дата регистрации в МЮ

·         Дата включения в версию

·         Вид правовой информации

·         Значимость

·         Территория

·         Принадлежность к информационным сегментам

·         Орган

·         Размер документа в байтах

См. также Полный список атрибутов документа

2.48     СПП

См. Сервер пакетного пополнения (СПП)

2.49     Тематически близкий документ

То же, что и Аналогичный документ.

2.50     Термин

Атрибут документа “Толкового Словаря”. Представляет собой совокупность слов, идентифицирующих понятие, на заданном языке.

2.51     Толкование

Атрибут документа “Толкового Словаря”. Представляет собой расширенное описание понятия на заданном языке.

2.52     ТС

Продукт компании Гарант-Сервис “Толковый Словарь”

2.53     Элемент словаря

Значение, содержащееся в словаре, которое может принять реквизит документа.
