Действие: Создайте Архитектурное доказательство концепции 

Activity: Construct Architectural Proof-of-Concept

Цель  

Purpose 

  • Синтезировать по крайней мере одно решение (которое может быть просто концептуальным), которое отвечает критическим архитектурным требованиям.
  • To synthesize at least one solution (which may simply be conceptual) that meets the critical architectural requirements.

Выберите Конструктивный подход

Decide on Construction Approach

Выберите методы которые будут использоваться для создания Архитектурного доказательства концепции, например:

Select the techniques to be used for construction of the Architectural Proof-of-Concept, for example:

  • Концептуальное моделирование
  • 'Быстрое' макетирование
  • Имитационное Моделирование
  • Автоматический перевод спецификаций для кодирования
  • 'Выполнимые' спецификации
  • Создание "пиков" как опытных прототипов - вертикальные вырезки через слои
  • Conceptual modeling
  • 'Rapid' Prototyping
  • Simulation
  • Automatic translation of specifications to code
  • 'Executable' specifications
  • Construction of  'spikes' as prototypes - vertical slices through layers

Создатель программного обеспечения должен быть способным рассуждать об этих моделях, в процессе обнаружения чего-либо, касающегося проблемы и областей решения.

The software architect needs to be able to reason about these models, in the process discovering something about both problem and solution spaces.

Выберите Активы и Технологии для Архитектурного Доказательства Концепции

Select Assets and Technologies for Architectural Proof-of-Concept  

Создатель программного обеспечения должен выбрать из активов и технологий, определенных  в разделе Действие: Архитектурный Анализ, те, которые будут использоваться при создании Архитектурного доказательства концепции.

The software architect should select, from the assets and technologies identified in Activity: Architectural Analysis, those to be used in the construction of the Architectural Proof-of-Concept.

Создайте Архитектурное Доказательство Концепции

Construct Architectural Proof-of-Concept  

Создатель программного обеспечения строит Архитектурное доказательство концепции, используя отобранные для создания методы, активы и технологии для того, чтобы соответствовать степени, требуемой профилем риска проекта - архитектурно значимым требованиям как записано в стереотипной реализации прецедента, обзоре проекта, развертываемых моделях и документе программной архитектуры.

Using the techniques selected for construction, the software architect builds the Architectural Proof-of-Concept, using the selected assets and technologies, to satisfy - to the extent required by the risk profile of the project - the architecturally significant requirements as captured in stereotypical use-case realizations, the overview design and deployment models, and the software architecture document.
