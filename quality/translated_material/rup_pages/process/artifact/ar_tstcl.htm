Цель

Purpose

Цель тест-класса состоит в том, чтобы определить специфические функциональные возможности теста, облегчить или автоматизировать тесты. Эти тест-специфичные функциональные возможности должны быть включены в проектную модель. Имеются два основных типа тест-специфичных  классов в проектной модели:

The purpose of the test class is to identify the test specific functionality to facilitate or automate tests. This test-specific functionality should be included in the design model. There are two main types of test-specific classes in a design model:

  • Тест-классы, являющиеся "обломками" проектных классов, которые Вы должны или смоделировать, или решить не включать полностью в цель теста.
  • Тест-классы, которые являются драйверами, облегчающими автоматизацию тест-установок.
 
  • Test classes that are "stubs" for design classes that you need to simulate or have decided not to include completely in a test target.
  • Test classes that are drivers, facilitating the automation of test cases.

Реализатор использует тест-классы, чтобы осуществить специфические функциональные возможности.

The implementer uses the test classes to implement the test specific functionality.

Test classes are created and modified in parallel with creating and modifying corresponding design classes.
