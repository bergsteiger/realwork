Цель

Purpose

Цель тест-компонента состоит в том, чтобы осуществить специфические функциональные возможности теста, направленные на облегчение или автоматизирование тестов. Имеются два основных типа тест-специфичных компонентов:

The purpose of the test component is to implement the test specific functionality to facilitate or automate tests. There are two main types of test-specific components:

  • Тест-компоненты, являющиеся "обломками" проектных классов, которые Вы должны смоделировать или решить не включать полностью в цель теста.
  • Тест-компоненты, которые являются драйверами, облегчающими автоматизацию тест-установок.

 

  • Test components that are "stubs" for design classes that you need to simulate or have decided not to include completely in a test target.
  • Test components that are drivers, facilitating the automation of test cases.

 

Испытатель (тестер) использует тест-компоненты для выполнения специфических функциональных возможностей при тестировании.

The Tester uses the test components to execute the test specific functionality in testing.
