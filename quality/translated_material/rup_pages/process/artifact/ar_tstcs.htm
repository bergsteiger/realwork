Тест-Установки

Test Case

 Тест-Установки - набор входных данных теста, условий выполнения, и ожидаемых результатов, разработанный для конкретной цели, такой как отработка определенной программной цепи или проверка соответствия специфическому требованию.

A test case is a set of test inputs, execution conditions, and expected results developed for particular objective, such as to exercise a particular program path or to verify compliance with a specific requirement

Цель

Purpose  

Цель Тест-Установок состоит в том, чтобы определить и связать условия, которые будут реализованы в тесте и которые являются необходимыми для проверки успешной и приемлемой реализации требований к продукту (прецеденты, производительность, и т.д.). Тест-Установки отражают один показатель тестового покрытия (множества неисправностей), а именно - основанного на требованиях, в котором каждая тест-установка происхождит по крайней мере от одного требования к тесту - что отражает требования к продукту, (см. Концепции: Ключевые Показатели Тестирования).

The purpose of the Test Case is to identify and communicate the conditions which will be implemented in test and are necessary to verify successful and acceptable implementation of the product requirements (use cases, performance characteristics, etc.). Test cases reflect one measure of test coverage, requirements-based test coverage, in that each test case traces back to at least one requirement for test - which reflect the product requirements, (see Concepts: Key Measures of Testing).

Следующие исполнители используют Тест-Установки:

The following workers use the Test Case:

  • Проектировщик Теста , определяющий тест-процедуры и тест-сценарии.
  • Проектировщик ответственный за проект.
  • Реализатор ответственный за реализацию компонентов и функциональных возможностей теста.
  • The Test Designer to identify test procedures and test scripts.
  • The Designer as input to design.
  • The Implementer as input for implementing components and test functionality.

 

 

Краткое Описание

Brief Outline

1. Test Case Description

A description of the condition, program path, or objective that this set of data implements / executes.

2. Test Inputs

The objects or fields the actor interacts with and the specific data values entered (or object states created) by the actor when executing this test case.

3. Expected Results

The resulting state or data received upon completion of executing this test case.

Согласование по времени

Начальные Тест-Установки определяются в процессе проектировки теста в Разработке, по мере того как определяются определенные прецеденты или сценарии. Тест-Установки совершенствуются на протяжении оставшейся части жизненного цикла в течение каждой итерации.

The initial Test Cases are identified during the test design activity in Elaboration as soon as there are some use cases or scenarios defined. The Test Cases are refined throughout the remainder of the lifecycle during each iteration.

Ответственность

Responsibility

Разработчик Теста ответствен за целостность Тест-Установок, гарантируя что:

A Test Designer is responsible for the integrity of the Test Cases, ensuring that:

  • Тест-Установки точно отражают содержание итерации.
  • Тест-Установки имеют соответствующее содержание и детализируют что необходимо для создания условий выполнения тест-установок, и оценивают его выполнение.
  • Для каждой Тест-Установки выясняется происхождение по крайней мере одного требования к тесту.
  • Достаточные Тест-Установки были определены для проверки удовлетворительного достижения требований к продукту.

 

  • The Test Cases accurately reflects the contents of the iteration.
  • The Test Cases contain the appropriate content and detail necessary to create a condition execute the test case, and evaluate its execution.
  • Each Test Case is traced backed to at least one requirement for test.
  • Sufficient Test Cases have been identified to verify satisfactory achievement of the product requirements.

Адаптация

Tailoring  

Тест-Установки (содержание и формат) могут нуждаться в модификации для достижения требований внутренних или внешних стандартов, руководств, или контроля.
