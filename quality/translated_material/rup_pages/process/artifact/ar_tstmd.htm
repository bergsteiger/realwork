Тест-Модель

 

 

 

Test Model

 Тест-модель является представлением того, что будет тестировано и каким образом это будет сделано. Это - вид проекта и моделей реализации, который характеризует непосредственно сам тест,  а также аспекты цели теста важные для проведения тестирования. Она включает набор тест-установок, тест-процедур, тест-сценариев, и ожидаемых результатов теста наряду с описанием их отношений.

The test model is a representation of what will be tested and how it will be tested. It is a view of the design and implementation models, depicting the tests themselves, as well as aspects of the target-of-test that are relevant to the testing effort. It includes the collection of the test cases, test procedures, test scripts, and expected test results along with a description of their relationships.

Цель

Purpose

Цель Тест-модели состоит в том, чтобы определить что тестируется, выразить возможности тестирования, и как тест-установки реализуются тест-процедурами. Тест-модель используют следующие люди:

The purpose of the test model is to identify what is being tested, express the scope of testing, and how the test cases are implemented by test procedures. The following people use the test model:

  • Разработчик теста использует тест-модель для того, чтобы выразить возможности теста и показать как тест-установки реализуются тест-процедурами.
  • Тестер использует тест-модель для того, чтобы понять цели теста.
  • Проектировщик использует тест-модель как исходные данные при проектировании  функциональных возможностей теста.
  • Реализатор использует тест-модель как исходные данные при реализации функциональных возможностей теста.

 

  • The test designer to express the scope of the tests and to show how test cases are implemented by test procedures.
  • The tester to understand the goals of test.
  • The designer as input to design of test functionality.
  • The implementer as input for implementing test functionality.

Тест-Процедуры

 

 

 

Test Procedures

 Набор детальных инструкций для установки, выполнения, и оценки результатов для данной тест-установки (или набора тест-установок).

A set of detailed instructions for the set-up, execution, and evaluation of results for a given test case (or set of test cases).

 
   
Тест-Установки

 

 

Test Cases

 Набор исходных данных теста, условий выполнения, и ожидаемых результатов, разработанных для специфической цели.

A set of test inputs, execution conditions, and expected results developed for particular objective.

 
   
Тест-сценарии

 

 

Test Scripts

 Машиночитаемый код, который автоматизирует выполнение тест-процедуры (или части тест-процедуры).

The machine readable code that automates the execution of a test procedure (or portion of a test procedure).

 
   

Согласование по времени

Timing

Первоначальная тест-модель создается в процессе проектирования теста в Разработке, по мере определения прецедентов или сценариев. Тест-модель совершенствуется на протяжении оставшейся части жизненного цикла в течение каждой итерации.

The initial test model is created during the test design activity in Elaboration, as soon as there are some use cases or scenarios defined. The test model is refined throughout the remainder of the lifecycle during each iteration.

Ответственность

Responsibility  

Разработчик Теста ответствен за целостность тест-модели, гарантируя что:

A Test Designer is responsible for the integrity of the test model, ensuring that:

  • Тест-модель точно отражает содержание итерации.
  • Тест-процедуры имеют соответствующее содержание и детализируют что необходимо для создания условий выполнения тест-установок, и оценивают его выполнение.
  • Достаточные Тест-Установки были определены для проверки удовлетворительного достижения прецедентов и/или требований.
  • The test model accurately reflects the contents of the iteration.
  • The test procedures contain the appropriate content and detail necessary to set-up, implement, execute, and evaluate the test cases.
  • Sufficient test cases have been identified to verify satisfactory achievement of the use cases and / or requirements.

Разработчик Тестов использует содержание тест-модели, чтобы произвести следующие имеющие к этому отношение артефакты:

The Test Designer uses the contents of the test model to produce the following related artifacts:

  • Тест-Процедуры
  • Прецеденты
  • Тест-Сценарии
  • Test Procedures
  • Test Cases
  • Test Scripts

Адаптация

Tailoring

Тест-Модель (содержание и формат) может нуждаться в модификации для достижения требований внутренних или внешних стандартов, руководств, или контроля.

The test model (contents and format) may require modification to meet the needs of internal or external standards, guidelines, or oversight.
