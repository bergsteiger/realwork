Цель

Purpose

Цель тест-пакета состоит в том, чтобы распознать и определить специфические функциональные возможности теста, направленные на облегчение или автоматизацию теста. Эти тест-специфичные функциональные возможности должны быть включены в проектную модель. Существует два основных типа тест-специфичных пакетов в проектной модели:

The purpose of the test package is to identify and define the test specific functionality to facilitate or automate tests. This test-specific functionality should be included in the design model. There are two main types of test-specific packages in a design model:

  • Тест-пакеты, являющиеся "обломками" проектных классов, которые Вы должны смоделировать или решить не включать полностью в цель теста.
  • Тест-пакеты, которые являются двигателями, облегчающими автоматизацию тест-установок.
  • Реализатор использует тест-пакеты для того, чтобы реализовать специфические функциональные возможности теста.

 

  • Test packages that are "stubs" for design classes that you need to simulate or have decided not to include completely in a test target.
  • Test packages that are drivers, facilitating the automation of test cases.
  • The Implementer uses the test packages to implement the test specific functionality.

Test packages are created and modified in parallel with creating and modifying corresponding design packages.
