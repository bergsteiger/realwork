Цель

Purpose

Результаты теста собираются во время проведения теста. Результаты теста используются в качестве исходных данных для оценки теста и для вычисления ключевых показателей теста.

Test results are captured during the execution of test. The test results are used as input for evaluating test and for calculating the key measures of test.   

Краткое описание

Brief Outline

Данные, содержащиеся в результатах теста могут меняться в зависимости от технологии и инструментов, используемых при проведении сбора данных в течение теста. Как минимум, следующие данные должны быть собраны и сделаны доступными для обзора и оценки:

The data contained by the test results may vary dependent upon the technology and tools used during test execution to capture the data.  At a minimum, the following data should be captured and made available for review and evaluation:

  • Идентификатор результата теста (ID для идентификации результатов этого теста от других)
  • Время, дата, ФИО тестера, и сопутствующая информация (такая как  операционная система, характеристики вычислительной машины, и т.д.)
  • Специфическая идентификация цели теста (такая как версия, объекты, файлы, и т.д.)
  • Тест-установки предназначенные  для выполнения (и которые должны быть соотнесены с требованиями к тесту)  
  • Выполненные тест-установки (и соотнесенные с требованиями к тесту)
  • Определение размера цели теста, которая должна быть выполнена
  • Определение размера   выполненной цели теста
  • Время ответа для специфических последовательностей событий
  • Зафиксированные данные, содержащие детали диалога между актером(ами) и целью теста, и/или между объектами в цели теста
  • Фактический результат каждой выполненной тест-установки
  • Различия между ожидаемым и фактическим результатом
  • Признак удачи/неудачи для каждой выполненной тест-установки
  • Любые неожидаемые или ненормальные результаты или линии поведения
  • test result identifier (ID for identifying these test results from others)
  • time, date, name of tester, and environment information (such as O/S, machine characteristics, etc.)
  • specific identification of the target-of-test (such as version, objects, files, etc.)
  • test cases intended to be executed (and traced to the requirements for test) 
  • test cases executed (and traced to the requirements for test)
  • size measurement of  target-of-test to be executed
  • size measurement of  target-of-test executed
  • response time for specified sequences of events
  • trace data containing the details of the conversations between actor(s) and the target-of-test, and / or between objects in the target-of-test
  • actual result of each test case executed
  • differences between expected result and actual result
  • an indication of pass / fail for each test case executed
  • any unexpected or abnormal results or behaviors

 

Согласование по времени

Timing

Результаты тестирования собираются в течение выполнения теста. Так как тест может проводиться много раз в течение жизненного цикла, результаты теста должны быть собраны и сохранены таким способом, что они могли быть рассмотрены и оценены индивидуально для каждого случая  проведения теста.

  The test results are captured during the execution of test. Since test execution may occur many times during the development lifecycle, test results should be captured and stored in such a way that they can be reviewed and evaluated individually for each instance of test execution.

Ответственность

Responsibility

Тестер ответствен за целостность артефакта результатов тестирования, гарантируя точность и полноту собранных данных.

A tester is responsible for the integrity of the test results artifact, ensuring that the data captured is accurate and complete.

Адаптация

Tailoring

Если Вы используете автоматизированный инструментарий теста, по типу того, который Вы можете найти в Rational Suite (Рациональном Наборе), или Test Studio (Студии Теста), большинство из вышеупомянутых данных и некоторые другие собираются и сохраняются автоматически. Инструментарий также предлагает сохраненные и настраиваемые представления данных для отчетных целей.
