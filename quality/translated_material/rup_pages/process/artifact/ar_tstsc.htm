Test Script

 Тест-сценарии - машиночитаемые инструкции, которые автоматизируют выполнение тест-процедуры (или ее части). Тест-сценарии могут создаваться (записываться) или производиться автоматически с помощью инструментария автоматизации теста, запрограммированного с использованием языка программирования, или комбинации записи, генерирования, и программирования.

Test scripts are the computer readable instructions that automate the execution of a test procedure (or portion of a test procedure). Test scripts may be created (recorded ) or automatically generated using test automation tools, programmed using a programming language, or a combination of recording, generating, and programming.

Цель

Purpose  

Цель Тест-Сценариев состоит в том, чтобы реализовывать и выполнять тест-процедуры рациональным и эффективным способом. Следующие люди используют тест-сценарии:

The purpose of the Test Scripts is to implement and execute the test procedures in an efficient and effective manner. The following people use the test scripts:

  • Разработчик Теста  -  чтобы создать тест-сценарии, которые выполняют тест-установки, достигают соответствующего тестового покрытия, максимизируют повторное использование, и устойчивость к  изменениям приложений.
  • Тестер -  чтобы выполнить тест и получить результаты.
  • The Test Designer to create test scripts that execute test cases, achieve the appropriate test coverage, maximize reuse, and are resilient to application changes.
  • The Tester to execute the tests and capture the results.

Инструкции

 

 

 

 

 

Instructions

 Машиночитаемые инструкции, при выполнении которых имитируются действия, осуществляемые актером при выполнении прецедента.

The machine readable instructions that, when executed, emulate the actions taken by the actor when executing the use case.

 

 Контрольная Точка 

 

 

 

Verification Point

 Место в инструкциях, где фактический результат будет сравнен с ожидаемым результатом.

The location in the instructions where an actual result will be compared with an expected result

 

Метод Проверки

 

 

 

 

 

 

Verification Method

 Машиночитаемая инструкция, которая выполняет сравнение между ожидаемыми и фактическими результатами. Существуют различные методы для проверки различных атрибутов.

The machine readable instruction that executes the comparison between expected and actual results. Different verification methods exist to verify different attributes.

 

Согласование по времени

Timing

Первичные Тест-сценарии создаются в течение реализации теста, по мере того как некоторые прецеденты или сценарии реализуются. Тест-сценарии изменяются и/или добавляются на протяжении оставшейся части жизненного цикла в течение каждой итерации.

The initial Test Scripts are created during the test implementation activity as soon as there are some use cases or scenarios implemented. The Test Scripts are modified and / or added to throughout the remainder of the lifecycle during each iteration.

 

Ответственность

Responsibility

Разработчик Теста ответствен за целостность Тест-Сценариев, гарантируя что:

A Test Designer is responsible for the integrity of the Test Scripts, ensuring that:

  • Тест-сценарии точно отражают тест-процедуры, которые будут осуществлены/выполнены (как в целых, так и в частичных процедурах) на основании тест-модели.
  • Тест-сценарии отражают выполнение прецедента, согласованное с эмулируемым актером (использование той же самой навигации, методов интерфейса и опций) или методы по максимизации эффективности и устойчивости тест-сценариев.
  • Тест-сценарии, если они были произведены автоматически, точно отражают структуру, и/или функциональную область, и/или UI цели испытания (или определенной части) и обеспечивают указанное тестовое покрытие.
  • The Test Scripts accurately reflect the test procedures to be implemented / executed (either in whole procedures or partial procedures) based upon the test model.
  • The Test Scripts reflect the execution of the use case consistent with the actor being emulated (using the same navigation, interface methods and options) or the methods to maximize the efficiency and resilience of the test scripts.
  • The Test Scripts, if automatically generated, accurately reflect the structure, and/or the functional area, and/or UI of the target-of-test (or specified portion) and provide the specified test coverage.

Адаптация

Tailoring

Тест-сценарии могут изменяться, чтобы включить концепции программирования, такие как переменные, циклы, и ветвление, направленные на то, чтобы увеличить эффективность, действенность, и устойчивость тест-сценариев.

The Test Scripts maybe modified to include a programming concepts, such as variables, loops, and branching to increase the efficiency, effectiveness, and resilience of the scripts.
