Цель

Purpose

Цель тест-подсистемы состоит в том, чтобы реализовать функциональные возможности теста, облегчить или автоматизировать тесты. Имеются два основных типа тест-специфичных подсистем:

The purpose of the test subsystem is to implement the test specific functionality to facilitate or automate tests. There are two main types of test-specific subsystems:

  • Тест-подсистемы, которые являются "обломками" для проектных классов, которые Вы должны смоделировать или решить не включать полностью в испытательную теста.
  • Тест-подсистемы, которые являются двигателями, облегчающими автоматизацию тест-установок.
  • Test subsystems that are "stubs" for design classes that you need to simulate or have decided not to include completely in a test target.
  • Test subsystems that are drivers, facilitating the automation of test cases.

 

Тестер использует Тест-подсистемы, чтобы реализовать функциональные возможности теста в тестировании. Тест-подсистема - своего рода подсистема реализации.

The Tester uses the test subsystems to execute the test specific functionality in testing. The test subsystem is a kind of implementation subsystem.
