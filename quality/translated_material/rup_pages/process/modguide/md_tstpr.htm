Тест-Процедура

 

Test Procedure

 Тест -процедура - набор детальных инструкций для установки, выполнения, и оценки результатов для данной тест-установки.

A test procedure is a set of detailed instructions for the set-up, execution, and evaluation of results for a given test case.

Объяснение

Explanation

Тест-процедура предоставляет детальные шаги, которые выполняют тест так, как определено соответствующей тест-установкой(ами).

A test procedure provides detailed steps that carry out the test as defined by the associated test case

Использование

Use

Проект тест-процедуры должен описывать следующие элементы:

The design of a test procedure needs to address the following items:

  • Предварительные условия для теста.
  • Исходные Условия
  • Действия, которые будут приняты.
  • Ожидаемые результаты.
  • Метод для проверки достоверности ожидаемых результатов.
  • pre-conditions for the test.
  • Input conditions.
  • Actions to be taken.
  • Expected results.
  • Method for validating expected results.

For example, if the test procedure is testing the use case or scenario for adding a customer account, the considerations might include:

  • Предварительные условия для теста - счет уже существует? Где в пределах программного обеспечения начинается тест-процедура? С выбора в меню Добавление Клиента? После выбора в меню  и когда уже открыт диалог Добавление Клиента?
  • Исходные Условия - тест-процедура обычно может тестировать правильность своей отправной точки, проверяя заголовок меню или диалога. Процедура затем вводит исходные данные, такие как имя и адреса клиента.
  • Действия, которые будут приняты, - процедура активирует  функцию "Сохранение".
  • Ожидаемые результаты - новый клиент должен быть сохранен в базе данных.
  • Метод для проверки достоверности ожидаемых результатов - Как процедура узнает, что клиент был добавлен? В этом случае, имеются много методов, включая метод, проверяющий, что новый клиент появляется при внесении в список счетов клиента, метод проверяющий подтверждающее сообщение, что клиент был добавлен, запрашивая непосредственно базу данных , и так далее.
  • pre-conditions for the test - Does the account already exist? Where within the software does the test procedure start? At a menu selection for Add Customer? After the menu selection with an Add Customer dialog already open?
  • Input conditions - The test procedure can usually test that its starting point is correct by checking a menu or a dialog box caption. The procedure then enters input data, such as customer name and address.
  • Actions to be taken - The procedure activates the "Save" function.
  • Expected results - The new customer should be saved in the database.
  • Method for validating expected results - How does the procedure know that the customer was added? In this case, there are many methods, including checking that the new customer appears in a listing of customer accounts, checking for a confirmation message that the customer was added, querying the database directly, and so on.
